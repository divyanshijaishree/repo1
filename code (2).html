Learn CPU Scheduling Algorithms

First Come First Serve

The simplest scheduling algorithm that executes processes in the order they arrive.

Shortest Job First

Prioritizes processes with the shortest execution time to minimize average waiting time.

Priority Scheduling

Executes processes based on priority levels assigned to each process.

Round Robin

Each process gets a small time slice of CPU time in a cyclic order.

Multilevel Queue

Divides processes into separate queues based on priority or process type.

Multilevel Feedback Queue

Allows processes to move between queues based on their behavior and CPU bursts.

Lottery Scheduling

Processes receive lottery tickets and the scheduler randomly picks a winning ticket.

Shortest Remaining Time

A preemptive version of SJF where the process with shortest remaining time runs next.

Fair Share Scheduling

CPU time is allocated fairly among users or groups rather than individual processes.

Earliest Deadline First

Processes with earliest deadlines are scheduled first, important for real-time systems.

Rate Monotonic

A static priority scheduling algorithm for periodic processes in real-time systems.

Longest Job First

Process with longest execution of time is selected for next execution.
